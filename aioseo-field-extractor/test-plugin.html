AIOSEO Plugin Field Structure Comparison

Old AIOSEO Plugin (v2.7.1)

SEO Title
Field name: _aioseop_title
Meta Description
Field name: _aioseop_description
Keywords
Field name: _aioseop_keywords (comma-separated)
Canonical URL
Field name: _aioseop_canonical_url
Custom Robots Meta
Field name: _aioseop_robots_meta
Old Plugin Data Structure:
{ "meta": { "_aioseop_title": "SEO Title", "_aioseop_description": "Meta Description", "_aioseop_keywords": "keyword1, keyword2, keyword3", "_aioseop_canonical_url": "https://example.com/canonical", "_aioseop_robots_meta": "index,follow" } }

New AIOSEO Plugin (v4.7.3+)

SEO Title
Field path: aioseo_meta_data.title
Meta Description
Field path: aioseo_meta_data.description
Focus Keyphrase
Field path: aioseo_meta_data.focus_keyphrase
Additional Keyphrases
Field path: aioseo_meta_data.keyphrases.additional
Canonical URL
Field path: aioseo_meta_data.canonical_url
New Plugin Data Structure:
{ "aioseo_meta_data": { "title": "SEO Title", "description": "Meta Description", "focus_keyphrase": "main keyword", "keyphrases": { "focus": { "keyphrase": "main keyword" }, "additional": [ {"keyphrase": "keyword1"}, {"keyphrase": "keyword2"} ] }, "canonical_url": "https://example.com/canonical" } }

Key Differences

  • Field Structure: Old plugin uses flat meta fields with "_aioseop_" prefix, new plugin uses nested "aioseo_meta_data" structure
  • Keywords: Old plugin combines all keywords in single "_aioseop_keywords" field, new plugin separates focus and additional keyphrases
  • API Endpoint: Both use WordPress REST API but with different field structures
  • Configuration: Set "seo_plugin_version": "old" or "new" in config to switch between formats
